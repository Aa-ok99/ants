# 🐜 AI Rules — @aa-ok99/ants

> กฎสำหรับ AI/Agent ทุกตัวที่เข้ามาทำงานในโปรเจกต์นี้

---

## 🎯 บุคลิก & สไตล์การทำงาน

1. **ใช้ภาษาไทยเสมอ** — ตอบเป็นภาษาไทย ยกเว้น code/commands ที่เป็นภาษาอังกฤษ
2. **กระชับ ตรงไปตรงมา** — ไม่ต้องอธิบายยาว ถ้าไม่จำเป็น
3. **ทำจริง ไม่พูดเปล่า** — ออกไอเดียแล้วลงมือทำเลย ไม่ต้องรอถาม
4. **ทำงานเป็นขั้นตอน** — แบ่งงานใหญ่เป็นขั้นตอนเล็ก ๆ ทำทีละขั้น
5. **ทดสอบก่อนส่ง** — รัน tests ทุกครั้งก่อน commit/push
6. **ไม่ทิ้งงานค้าง** — ถ้าเริ่มทำแล้ว ต้องทำให้เสร็จ หรือบอกสถานะชัดเจน

---

## 📋 กฎการทำงาน (สำคัญมาก)

### 1. ต้องทำ Todo List เสมอ

ทุกครั้งที่ได้รับงานใหม่ **ต้องสร้าง todo list ก่อน** แล้วอัปเดตสถานะตลอด:

```
- [ ] ขั้นตอนที่ 1
- [x] ขั้นตอนที่ 2 (เสร็จแล้ว)
- [ ] ขั้นตอนที่ 3
```

ใช้ `todowrite` tool เพื่อจัดการ todo list

### 2. ต้อง Push ไป GitHub ทุกครั้ง

หลัง commit แล้ว **ต้อง push ไป GitHub ทันที**:

```bash
git add -A && git commit -m "message" && git push origin main
```

**ห้าม commit แล้วไม่ push** — งานจะหายถ้า session จบ

### 3. ต้องรัน Tests ก่อน Publish

ก่อน publish ไป npm **ต้องรัน tests ก่อนเสมอ**:

```bash
node node_modules/jest/bin/jest.js --verbose
```

ถ้า tests ไม่ผ่าน **ห้าม publish**

### 4. ต้องใช้ภาษาไทยใน Commit Messages

Commit messages ควรเป็นภาษาอังกฤษ (ตาม convention) แต่ **อธิบายเป็นภาษาไทย** เมื่อสรุปให้ user

## 💡 สไตล์การออกไอเดีย

เมื่อออกไอเดียให้ user:

1. **จัดเป็นตาราง** — อ่านง่าย ชัดเจน
2. **เรียงตามความสำคัญ** — สำคัญสุดอยู่บน
3. **มีหมวดหมู่** — แบ่งเป็น bug fix, feature ใหม่, improvement
4. **บอกความยากง่าย** — ใช้ emoji หรือ label
5. **ทำเลย ไม่ต้องรอ** — ถ้า user บอก "ทำเลย" หรือ "ทั้งหมด" ให้เริ่มทำทันที

ตัวอย่าง:

```
## 💡 ไอเดียสำหรับ vX.X.X

| # | Feature | ความยาก | สถานะ |
|---|---------|---------|-------|
| 1 | แก้ bug A | 🟢 ง่าย | ✅ เสร็จ |
| 2 | เพิ่ม feature B | 🟡 ปานกลาง | 🔄 กำลังทำ |
| 3 | เพิ่ม feature C | 🔴 ยาก | ⏳ รอทำ |
```

---

## 🚀 Workflow ปกติ

1. **รับงาน** → สร้าง todo list
2. **วิเคราะห์** → ออกไอเดีย (ถ้าจำเป็น)
3. **ลงมือทำ** → ทำทีละขั้น อัปเดต todo ตลอด
4. **ทดสอบ** → รัน tests
5. **Commit & Push** → git add, commit, push
6. **Publish** → npm publish (ถ้าเป็น version ใหม่)
7. **สรุป** → บอก user ว่าทำอะไรเสร็จบ้าง

## 🚨 ห้ามทำ

- ❌ ห้าม commit แล้วไม่ push
- ❌ ห้าม publish โดยไม่รัน tests
- ❌ ห้ามทิ้งงานค้างโดยไม่บอกสถานะ
- ❌ ห้ามใช้ภาษาอังกฤษตอบ user (ยกเว้น code/commands)
- ❌ ห้ามแก้ไฟล์ที่ไม่เกี่ยวข้องกับงาน
- ❌ ห้ามลบ tests หรือลดจำนวน tests
